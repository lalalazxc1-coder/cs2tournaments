# Инструкция: Проверка и применение миграций на боевом сервере

## Проблема
На боевом сервере появляется сообщение:
```
ℹ️  Database schema managed by migrations. Run "npm run migrate" to update.
```

Это **не ошибка**, а информационное сообщение, которое показывается когда `NODE_ENV=production`.

## Как проверить что не загрузилось

### 1. Проверить статус миграций

```bash
cd /path/to/backend
npm run migrate:status
```

Вы увидите список миграций:
- `up` - миграция применена ✅
- `down` - миграция НЕ применена ❌

### 2. Применить недостающие миграции

```bash
npm run migrate
```

### 3. Проверить индексы в базе данных

Используйте скрипт для проверки:

```bash
node scripts/check-indexes.js
```

## Типичные проблемы

### Ошибка: "Duplicate key name"

Если миграция пытается создать индекс, который уже существует:

1. **Причина**: Индекс был создан вручную или другой миграцией
2. **Решение**: Обновлена миграция `20251215174014-add-performance-indexes.js` с проверкой существования индексов

### Недостающие миграции

Если `npm run migrate:status` показывает миграции со статусом `down`:

```bash
# Применить все недостающие миграции
npm run migrate

# Откатить последнюю миграцию (если что-то пошло не так)
npm run migrate:undo
```

## Команды для работы с миграциями

| Команда | Описание |
|---------|----------|
| `npm run migrate` | Применить все новые миграции |
| `npm run migrate:status` | Показать статус миграций |
| `npm run migrate:undo` | Откатить последнюю миграцию |
| `npm run migrate:undo:all` | Откатить все миграции |
| `npm run migration:create` | Создать новую миграцию |

## Проверка работоспособности

После применения миграций проверьте:

1. **Health Check**:
   ```bash
   curl http://your-server:5000/api/health
   ```

2. **Логи сервера**:
   ```bash
   pm2 logs
   ```

3. **Индексы в базе**:
   ```bash
   node scripts/check-indexes.js
   ```

## Различия между тестом и боевым сервером

| Параметр | Тест (development) | Боевой (production) |
|----------|-------------------|---------------------|
| `NODE_ENV` | `development` | `production` |
| Sync моделей | `sequelize.sync()` ✅ | Отключено ❌ |
| Миграции | Опционально | **Обязательно** |
| Сообщение | Не показывается | Показывается |

**Важно**: На боевом сервере схема БД управляется **только через миграции**, автоматическая синхронизация отключена для безопасности данных.
